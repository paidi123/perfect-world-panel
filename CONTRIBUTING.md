# Contributing to Perfect World Panel

Thank you for your interest in contributing to Perfect World Panel! This document provides guidelines and instructions for contributing.

## Code of Conduct

Please be respectful and inclusive in all interactions.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or fix
4. Make your changes
5. Write or update tests as needed
6. Push to your fork
7. Submit a pull request

## Development Setup

See README.md for detailed setup instructions.

## Code Style

### PHP
- Follow PSR-12 coding standard
- Use meaningful variable and function names
- Add comments for complex logic
- Use type hints

### JavaScript/Vue
- Use ESLint for code linting
- Follow Vue.js style guide
- Use meaningful component names
- Add comments for complex logic

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove, etc.)
- Keep messages concise but informative
- Reference issues when applicable

Example:
```
Fix: Resolve account ban logic issue (#123)
Add: Implement character level reset functionality
```

## Pull Requests

1. Provide a clear description of your changes
2. Reference any related issues
3. Ensure all tests pass
4. Update documentation if needed
5. Keep PRs focused and manageable

## Testing

- Write tests for new features
- Ensure all existing tests pass
- Aim for high code coverage
- Test edge cases

## Reporting Issues

When reporting issues, please include:
- Clear description of the problem
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details (OS, PHP version, etc.)

## Feature Requests

Feature requests are welcome! Please describe:
- The feature you'd like
- Why you need it
- How it should work
- Any alternative approaches

Thank you for contributing!
